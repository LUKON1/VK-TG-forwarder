# VK → TG Forwarder

Пересылает сообщения и фотографии из VK групповой беседы в Telegram-группу.

## Как работает

VK Community Bot (сообщество) добавляется в беседу как участник с правом читать все сообщения. Бот слушает события через **Bots Long Poll API** и пересылает их в TG через Bot API.

```
VK Беседа → Bots Long Poll → Node.js → Telegram Bot API → TG Группа
```

**Формат сообщений в TG:** `[Имя Фамилия]: текст`

## Требования

- Node.js >= 20
- Docker + Docker Compose (для деплоя)
- VK-сообщество (любого типа)
- Telegram-бот от @BotFather

## Настройка VK-сообщества (один раз)

**1. Включить сообщения и бота**  
Управление → Сообщения → включить  
Управление → Сообщения → Настройки для бота → включить **"Разрешать добавлять сообщество в беседы"**

**2. Получить токен**  
Управление → Дополнительно → Работа с API → Создать ключ  
Выбрать право: **Сообщения** → скопировать токен

**3. Включить Long Poll**  
Управление → Дополнительно → Работа с API → Long Poll API → включить

**4. Добавить сообщество в беседу**  
На странице сообщества → "Добавить в чат" → выбрать целевую беседу

**5. Дать права на чтение всех сообщений**  
В беседе → Участники → сообщество → **Доступ ко всей переписке**  
*(делает любой администратор беседы)*

## Конфигурация

Скопируй `.env.example` → `.env` и заполни:

```env
VK_COMMUNITY_TOKEN=  # токен из шага 2
VK_PEER_ID=          # из URL беседы: vk.com/im/convo/2000000007 → 2000000007
TG_BOT_TOKEN=        # от @BotFather
TG_CHAT_ID=          # ID TG-группы (можно узнать через @userinfobot)
```

## Запуск

### Локально
```bash
npm install
npm start
```

### Docker (на сервере)
```bash
# Скопировать проект на сервер вместе с .env
docker compose up --build -d
```

Контейнер автоматически поднимается при перезагрузке (`restart: unless-stopped`).

## Когда нужно вмешаться

Только если токен сообщества был вручную отозван в панели управления.  
В остальном — работает автономно без ограничений по времени.
